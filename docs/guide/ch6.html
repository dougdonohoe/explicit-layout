Why nest containers?

With less capable layout managers, it is sometimes necessary to nest containers to achieve a desired layout. Nesting containers involves grouping components into invisible containers, possibly using a different layout manager for each container and adding each container, rather than the individual components, to the main container. The need to nest containers for this reason is reduced when using ExplicitLayout because it provides more flexible control over the layout.

However, there are many other reasons why you may want to group components in a container including:

  • Improving modularity or the code
  • Better object orientated design
  • Maintainability of the code
  • To make a group of components invisible
  • To add components to a scroll pane

Container size hints

It is typical for a layout manager to define the preferred, minimum and maximum size of the container based on the layout components and constraints. These size hints may be used by the layout manager of the parent container when nesting containers.

ExplicitLayout does not automatically define these size hints based on the layout. Instead, it allows you to specify expressions for the preferred, minimum and maximum sizes. These expressions can be defined by combining expressions from the GroupEF, ComponentEF and MathEF expression factories. These expressions could depend on some or all of the components in the container; alternatively they may be set to a constant expression.

The following code specified the preferred size of the container. The preferred width is set to be the maximum preferred width of a group of components plus some space. The preferred height is set to the sum of the preferred heights of a group of components plus some space.

ExplicitLayout layout = new ExplicitLayout();
Component [] group = {label1, label2, label3, label4};
Expression width = MathEF.max(GroupEF.createExpressions(ComponentEF.preferredWidth(null), group)); Expression height = MathEF.add(GroupEF.preferredWidth(group), (group.length + 1) * 5);
layout.setPreferredSize(width, height);
